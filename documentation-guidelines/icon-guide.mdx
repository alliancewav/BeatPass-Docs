# Icon & Visual Standards

Consistent visual language for release notes to improve scannability and recognition.

<Warning>
  **Icon Library: Font Awesome 6 ONLY.** The BeatPass docs are configured with `"icons": { "library": "fontawesome" }` in `docs.json`. Never use Lucide, Heroicons, or any other library's icon names. Verify every icon at [fontawesome.com/icons](https://fontawesome.com/icons) before use.
</Warning>

## Usage Patterns

### In Cards

```mdx
<Card title="Feature Name" icon="star" color="#f59e0b">
  Description of the feature
</Card>
```

### In Accordions

```mdx
<Accordion title="Feature Category" icon="bolt">
  Content here
</Accordion>
```

### In Updates

```mdx
<Update label="Date" tags={["feature"]} icon="star">
  Update content
</Update>
```

---

## Color Accessibility

All color combinations meet WCAG AA standards:
- Dark text on light backgrounds
- Icons use high-contrast colors
- Avoid color-only indicators (always pair with icon or text)

---

### Major Release Card
