## Overview

The BeatPass API provides programmatic access to platform data and functionality. This documentation covers public API capabilities for approved integrators.

<Warning>
  The API is currently in beta. Endpoints may change. Contact us before building integrations.
</Warning>

---

## Base URL

All API requests use the base URL:

```
https://beta.beatpass.ca/api/v1
```

---

## API Capabilities

### Available Features

| Capability | Description |
|------------|-------------|
| Track Data | Access public track information |
| Artist Profiles | Retrieve artist and producer data |
| Search | Query tracks and artists |
| User Data | Access authenticated user information |

### Coming Soon

- Webhook notifications
- Bulk data exports
- Analytics API
- Upload API

---

## Request Format

### HTTP Methods

| Method | Usage |
|--------|-------|
| GET | Retrieve data |
| POST | Create resources |
| PUT/PATCH | Update resources |
| DELETE | Remove resources |

### Headers

Required headers for all requests:

```http
Accept: application/json
Content-Type: application/json
```

For authenticated endpoints:

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

---

## Response Format

### Success Response

```json
{
  "status": "success",
  "data": {
    // Response data
  }
}
```

### Error Response

```json
{
  "status": "error",
  "message": "Error description",
  "errors": {
    // Field-specific errors if applicable
  }
}
```

### Pagination

Paginated endpoints return:

```json
{
  "data": [...],
  "pagination": {
    "current_page": 1,
    "per_page": 15,
    "total": 150,
    "last_page": 10
  }
}
```

---

## Common Endpoints

### Tracks

```http
GET /tracks              # List tracks
GET /tracks/{id}         # Get track details
GET /tracks/{id}/stream  # Stream URL
```

### Artists

```http
GET /artists             # List artists
GET /artists/{id}        # Get artist details
GET /artists/{id}/tracks # Artist's tracks
```

### Search

```http
GET /search?query=term   # Search tracks and artists
```
